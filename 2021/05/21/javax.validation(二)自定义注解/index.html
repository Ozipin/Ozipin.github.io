javax.validation(二)自定义注解

发表于 2021-05-21 | 分类于 java
利用javax.validation机制，扩展自定义的验证注解

​ 注：本文展示的代码或有不全，代码详情可见github: https://github.com/Ozipin/file-master

​ 上篇文章我们介绍了javax.validation数据格式验证的基本用法，也知道了javax.validation其实只是定义了验证的注解，具体的实现是依赖于其他的jar包，如我们上文的代码示例都是依赖于hibernate-validator。

​ javax.validation的22个注解在绝大多数时候都是可以满足我们对于数据格式验证的需要的，但是由于实际业务的需求，还是会对与数据格式的验证有一些比较特殊的需求，而现有的注解又无法满足的，这个时候我们的做法通常是编写业务代码来专门处理这些验证，但是在我们了解了javax.validation的设计和其实现之后，我们也可以尝试进行自己的验证注解定义和实现。

​ 比如：我们的项目中可能会用到用户的地址和手机号码，但是公司的安全规定则是要求这些敏感数据需要加密传输，所以我们后端接收到的前端传参一定会是利用了指定公钥进行了加密的数据，而我们需要对数据进行解密以及验证。（实际上正则表达式的验证注解可以满足我们的绝大部分的字符串字段的验证）
